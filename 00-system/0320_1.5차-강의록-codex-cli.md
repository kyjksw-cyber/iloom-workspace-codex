## Claude Code vs Codex CLI — 차이점

| | Claude Code | Codex CLI |
|--|-----------|-----------|
| 만든 곳 | Anthropic | OpenAI |
| AI 모델 | Claude | GPT-4o / o4-mini |
| 핵심 설정 파일 | `CLAUDE.md` | `AGENTS.md` |
| 슬래시 커맨드 | 있음 (`/daily-note` 등) | 없음 — 자연어로 요청 |
| 스킬 (자동 트리거) | 있음 | 없음 — AGENTS.md에 통합 |
| 메모리 | 자동 (세션 간 기억) | 없음 — 파일로 직접 관리 |

> 도구는 다르지만 **원리는 같습니다.**
> 설정 파일에 맥락을 담고, 자연어로 지시하고, 결과를 기록으로 남긴다.

## AGENTS.md — Codex의 두뇌

Codex CLI는 매 세션마다 `AGENTS.md`를 자동으로 읽습니다.
여기에 적힌 규칙, 워크플로우, 브랜드 정보를 따릅니다.

**중요**: 하위 폴더에 `AGENTS.md`를 넣으면 그 폴더 작업 시 자동으로 추가 맥락이 됩니다.

```
iloom-workspace-codex/
├── AGENTS.md                    ← 전체 규칙 (항상 읽힘)
├── 10-projects/
│   └── 15-봄캠페인/
│       ├── AGENTS.md            ← 이 프로젝트 전용 맥락 (자동 읽힘)
│       └── progress.md
```

## 데일리 노트 만드는 법

Codex에게 자연어로 요청하면 됩니다:

```
> 데일리 노트 만들어줘
```

이렇게 하면 AGENTS.md에 적힌 워크플로우대로:
1. 오늘 날짜 확인
2. 템플릿(`00-system/01-templates/daily-note-template.md`) 읽기
3. `40-personal/41-daily/2026-03-20.md` 파일 생성

이미 있으면 내용 보여주고 추가 작성 여부를 물어봅니다.

### 작업 끝나고 기록 남기기

```
> 오늘 한 일 정리해서 데일리 노트에 추가해줘
```

## 맥락 파일 만드는 법 (progress.md)

### 왜 필요한가

Codex는 **메모리가 없습니다.** 대화창을 닫으면 전부 잊어버립니다.
그래서 프로젝트마다 `progress.md`를 만들어서 **"여기까지 했어"** 를 남겨야 합니다.

### 프로젝트 구조

```
10-projects/15-봄캠페인/
├── AGENTS.md        ← "이 프로젝트는 이런 거야" (Codex 자동 인식)
├── progress.md      ← "여기까지 했어" (매 작업 후 업데이트)
└── memo.md          ← 작업 메모
```

### 만드는 법

```
> 새 프로젝트 만들어줘. 이름: 봄 SNS 캠페인
```

자동으로 AGENTS.md + progress.md + memo.md가 생깁니다.

### 프로젝트 AGENTS.md 예시

```markdown
# 봄 SNS 캠페인

## 현재 상태
- 기획 완료, 콘텐츠 제작 중

## 핵심 맥락
- 목표: 봄 신제품 인지도 확보
- 타겟: 2030 신혼부부
- 기한: 2026-04-15

## 작업할 때 읽어야 할 파일
- progress.md — 진행 현황
```

### progress.md 예시

```markdown
# 봄 SNS 캠페인 Progress

## 최근 업데이트
- 2026-03-20: 인스타 3건 초안 완성

## 완료
- [x] 캠페인 기획서 작성
- [x] 인스타 포스트 3건 초안

## 진행 중
- [ ] 블로그 포스트 1건

## 다음 할 일
- [ ] 콘텐츠 팀 리뷰
```

### 이어서 작업할 때

다음 세션에서:
```
> 봄 캠페인 이어서 할게. progress.md 먼저 읽어줘.
```

→ Codex가 progress.md를 읽고 맥락을 파악한 상태에서 작업합니다.

### 작업 끝나면

```
> progress.md 업데이트해줘
```

---

## 왜 이게 중요한가

### 1. AI와의 대화는 휘발된다

Codex와 좋은 작업을 했어도, 세션을 닫으면 **백지**입니다.
데일리 노트와 progress.md는 **내가 뭘 했는지, 다음에 뭘 해야 하는지** 남기는 유일한 방법입니다.

### 2. 기록은 쌓이면 자산이 된다

하루치 데일리 노트는 별거 아닙니다.
**한 달치가 쌓이면:**

- "이번 달 내가 한 일 정리해줘" → 월간 리포트 자동 생성
- "지난달에 비슷한 작업 했었어?" → 과거 맥락 검색
- "내 업무 패턴 분석해줘" → 업무 효율 인사이트

> **"기록이 목표가 되면 일이 돼버려요. 일의 부산물로 알아서 나오게 하는 게 핵심."**

데일리 노트는 **1분이면 됩니다.** AI한테 시키면 되니까.

### 3. 습관이 곧 역량이다

AI 에이전트를 잘 쓰는 사람의 공통점:
1. 매일 기록한다 — 데일리 노트
2. 맥락을 넘긴다 — progress.md
3. 패턴을 만든다 — 반복 워크플로우

### 요약

```
┌─────────────────────────────────────────────┐
│  AGENTS.md (루트)                            │
│  → 전체 규칙, "progress.md 먼저 읽어"         │
├─────────────────────────────────────────────┤
│  프로젝트/AGENTS.md                          │
│  → 이 프로젝트의 목표, 맥락                   │
├─────────────────────────────────────────────┤
│  프로젝트/progress.md                        │
│  → 여기까지 했고, 다음은 이거                  │
├─────────────────────────────────────────────┤
│  40-personal/41-daily/YYYY-MM-DD.md          │
│  → 오늘 한 일, 내일 할 일                     │
└─────────────────────────────────────────────┘

이 4개를 관리하면 → Codex가 항상 맥락 있게 일합니다.
```

## 숙제: 데일리 노트 5일 챌린지

### Step 1: 워크스페이스 다운로드

```bash
git clone https://github.com/kyjksw-cyber/iloom-workspace-codex.git
cd iloom-workspace-codex
```

### Step 2: AGENTS.md 본인 정보로 수정

파일 상단의 사용자 프로필을 본인 정보로 바꾸세요:

```markdown
## 사용자 프로필
- **이름**: (본인 이름)
- **소속**: (본인 팀/역할)
- **주요 업무**: (본인 업무)
```

### Step 3: 매일 데일리 노트 만들기

Codex를 열고:
```
> 데일리 노트 만들어줘
```

최소 **"오늘 한 일"** 과 **"내일 할 일"** 두 섹션만 채우면 OK.

### Step 4: 5일치 완성

```
40-personal/41-daily/
├── 2026-03-24.md
├── 2026-03-25.md
├── 2026-03-26.md
├── 2026-03-27.md
└── 2026-03-28.md
```

### 제출

5일치를 GitHub에 push하세요:
```bash
git add -A
git commit -m "데일리노트 5일 챌린지"
git push
```
