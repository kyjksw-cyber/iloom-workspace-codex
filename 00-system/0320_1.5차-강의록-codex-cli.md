## 도입 (10분) — 왜 두 번째 도구를 배우는가

### 이야기할 것

1차에서 Claude Code를 써봤습니다. 잘 작동하죠.
그런데 **왜 또 다른 도구를 배워야 할까요?**

#### 이유 1: AI 에이전트는 하나가 아니다

| 도구 | 만든 곳 | AI 모델 |
|------|--------|---------|
| Claude Code | Anthropic | Claude |
| Codex CLI | OpenAI | GPT-4o, o4-mini |
| Cursor, Windsurf, Copilot... | 각각 | 다양 |

→ 특정 도구에 종속되면 안 된다. **원리를 알면 어떤 도구든 쓸 수 있다.**

#### 이유 2: 도구마다 잘하는 게 다르다

| 상황 | 더 나은 선택 |
|------|------------|
| 복잡한 자동화 파이프라인 | Claude Code (스킬, MCP, 서브에이전트) |
| 빠르게 파일 작업 | Codex CLI (심플, 빠름) |
| 코드 작성/리팩토링 | 둘 다 OK |

#### 이유 3: 핵심 원리는 같다

**모든 AI 에이전트의 작동 원리:**
```
설정 파일을 읽는다 → 내 지시를 따른다 → 내 폴더에서 작업한다
```

- Claude Code: `CLAUDE.md`를 읽는다
- Codex CLI: `AGENTS.md`를 읽는다
- 이름만 다를 뿐 **원리는 동일**

> **"도구를 배우는 게 아니라, AI 에이전트와 일하는 법을 배우는 겁니다."**


## Part 2 (15분) — Claude Code vs Codex, 뭐가 다른가

### 핵심 슬라이드: 설정 파일 비교

```
Claude Code                    Codex CLI
─────────────                  ──────────
CLAUDE.md        ←  같은 역할  →  AGENTS.md
.claude/commands/  ←  슬래시 커맨드  →  (없음, 자연어로)
.claude/skills/    ←  자동 트리거    →  (없음, AGENTS.md에 통합)
.claude/memory/    ←  자동 기억      →  (없음, progress.md로)
```

### 데모: 같은 작업, 다른 방식

**"데일리 노트 만들기"**

| Claude Code | Codex CLI |
|------------|-----------|
| `/daily-note` 입력 | "데일리 노트 만들어줘" 입력 |
| 커맨드가 자동 실행 | AGENTS.md 워크플로우 따라 실행 |
| 결과 동일 | 결과 동일 |

> 💬 "보세요, 결과는 같습니다. 방법만 다를 뿐."

### 핵심 슬라이드: 기능 차이 정리

| | Claude Code | Codex CLI |
|--|-----------|-----------|
| 장점 | 자동화 극대화 (스킬, 메모리, MCP) | 심플, 배우기 쉬움, 빠름 |
| 단점 | 설정이 복잡 | 자동화 기능 적음 |
| 맥락 유지 | 자동 (메모리) | 수동 (progress.md) |
| 추천 상황 | 반복 자동화, 복잡한 파이프라인 | 빠른 파일 작업, 코드 작업 |

### 핵심 메시지

> **"Codex가 기능이 적은 게 아니라, 다른 철학입니다."**
> - Claude Code: "자주 하는 걸 자동화하자" → 스킬, 커맨드, 메모리
> - Codex CLI: "설정 파일 하나로 다 하자" → AGENTS.md에 집중

## Part 3 (15분) — 맥락 있이 일하는 법

### 문제 제기

> 💬 "Codex는 메모리가 없습니다. 대화창을 닫으면 다 잊어버려요."
> "그러면 어떻게 프로젝트를 이어서 하죠?"

### 해결: 3층 구조

```
┌─────────────────────────────────────┐
│  1층: AGENTS.md (루트)               │
│  → 전체 규칙, 활성 프로젝트 목록      │
├─────────────────────────────────────┤
│  2층: 프로젝트/AGENTS.md             │
│  → 이 프로젝트의 목표, 맥락, 상태     │
├─────────────────────────────────────┤
│  3층: 프로젝트/progress.md           │
│  → 여기까지 했고, 다음은 이거         │
└─────────────────────────────────────┘
```

### 라이브 데모

**1. 새 프로젝트 만들기**
```
> 새 프로젝트 만들어줘. 이름: 봄 SNS 캠페인
```

**2. 작업하기**
```
> 인스타 포스트 초안 3개 만들어줘. 주제는 봄 신제품 책상.
```

**3. progress.md 업데이트**
```
> progress.md 업데이트해줘
```

**4. 세션 종료 → 새 세션에서 이어하기**
```
> 봄 SNS 캠페인 이어서 할게. progress.md 먼저 읽어줘.
```

### 핵심 메시지

> **"AI한테 기억력을 기대하지 마세요. 대신 기록을 남기세요."**
> 이건 Codex만이 아니라 **모든 AI 에이전트와 일할 때의 원칙**입니다.
