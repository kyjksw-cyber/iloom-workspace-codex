# Part 1. 개념 — AI 에이전트와 일하는 원리

---

## Claude Code vs Codex CLI

1차에서 Claude Code를 배웠습니다. 이번엔 OpenAI의 Codex CLI를 씁니다.
도구는 다르지만 **원리는 같습니다.**

| | Claude Code | Codex CLI |
|--|-----------|-----------|
| 만든 곳 | Anthropic | OpenAI |
| AI 모델 | Claude | GPT-4o / o4-mini |
| 핵심 설정 파일 | `CLAUDE.md` | `AGENTS.md` |
| 슬래시 커맨드 | 있음 (`/daily-note` 등) | 없음 — 자연어로 요청 |
| 스킬 (자동 트리거) | 있음 | 없음 — AGENTS.md에 통합 |
| 메모리 | 자동 (세션 간 기억) | 없음 — 파일로 직접 관리 |

**공통 원리:**
1. 설정 파일에 맥락을 담는다 (CLAUDE.md / AGENTS.md)
2. 자연어로 작업을 지시한다
3. 결과를 기록으로 남긴다

이 원리를 알면 **어떤 AI 에이전트 도구가 나와도** 바로 적응할 수 있습니다.

---

## AGENTS.md — Codex의 두뇌

Codex는 매 세션마다 `AGENTS.md`를 자동으로 읽습니다.
Claude Code의 `CLAUDE.md`와 같은 역할입니다.

**하위 폴더에도 AGENTS.md를 넣을 수 있습니다** → 그 폴더 작업 시 자동으로 추가 맥락이 됩니다.

```
iloom-workspace-codex/
├── AGENTS.md                    ← 전체 규칙 (항상 읽힘)
├── 10-projects/
│   └── 15-봄캠페인/
│       ├── AGENTS.md            ← 이 프로젝트 전용 (자동 읽힘)
│       └── progress.md
```

---

## 왜 기록이 중요한가

### 1. AI와의 대화는 휘발된다

Codex와 좋은 작업을 해도, 세션을 닫으면 **전부 잊어버립니다.**
데일리 노트와 progress.md는 **"내가 뭘 했는지, 다음에 뭘 해야 하는지"** 남기는 유일한 방법입니다.

### 2. 기록은 쌓이면 자산이 된다

하루치는 별거 아닙니다. **한 달치가 쌓이면:**

- "이번 달 내가 한 일 정리해줘" → 월간 리포트 자동 생성
- "지난달에 비슷한 작업 했었어?" → 과거 맥락 검색
- "내 업무 패턴 분석해줘" → 업무 효율 인사이트

> **"기록이 목표가 되면 일이 돼버려요. 일의 부산물로 알아서 나오게 하는 게 핵심."**

데일리 노트는 **1분이면 됩니다.** AI한테 시키면 되니까요.

### 3. 습관이 곧 역량이다

AI 에이전트를 잘 쓰는 사람의 공통점:
1. 매일 기록한다 — 데일리 노트
2. 맥락을 넘긴다 — progress.md
3. 패턴을 만든다 — 반복 워크플로우

---

## 맥락 관리 구조

```
┌─────────────────────────────────────────────┐
│  AGENTS.md (루트)                            │
│  → 전체 규칙, "progress.md 먼저 읽어"         │
├─────────────────────────────────────────────┤
│  프로젝트/AGENTS.md                          │
│  → 이 프로젝트의 목표, 맥락                   │
├─────────────────────────────────────────────┤
│  프로젝트/progress.md                        │
│  → 여기까지 했고, 다음은 이거                  │
├─────────────────────────────────────────────┤
│  40-personal/41-daily/YYYY-MM-DD.md          │
│  → 오늘 한 일, 내일 할 일                     │
└─────────────────────────────────────────────┘
```

이 4개를 관리하면 → Codex가 항상 맥락 있게 일합니다.

# Part 2. 설치 — 하나씩 따라하기

---

## Step 1: Git 설치

Git은 파일 버전을 관리하는 도구입니다.
워크스페이스를 다운받고, 숙제를 제출하는 데 필요합니다.

### Windows

1. https://git-scm.com 접속
2. **"Download for Windows"** 클릭
3. 다운받은 설치 파일 실행
4. **모든 옵션 기본값(Next)** 으로 설치 — 아무것도 바꾸지 않아도 됩니다
5. 설치 완료 후 **시작 메뉴에서 "Git Bash" 검색** → 실행

설치 확인:
```bash
git --version
```
`git version 2.xx.x` 이런 식으로 나오면 성공!

### Mac

1. 터미널 열기 (Spotlight에서 "터미널" 검색)
2. 아래 입력:
```bash
git --version
```
3. 처음이면 "Xcode Command Line Tools 설치하겠냐"고 물어봄 → **설치** 클릭
4. 완료 후 다시 `git --version` → 버전 나오면 성공

---

## Step 2: Git 초기 설정

Git을 처음 쓰면 이름과 이메일을 등록해야 합니다.
**Git Bash** (Windows) 또는 **터미널** (Mac)에서:

```bash
git config --global user.name "홍길동"
git config --global user.email "gildong@fursys.com"
```

본인 이름과 회사 이메일로 바꿔서 입력하세요.

한글 파일명이 깨지지 않게 설정:
```bash
git config --global core.quotepath false
```

---

## Step 3: GitHub 계정 만들기

GitHub는 파일을 온라인에 저장하는 서비스입니다.

1. https://github.com 접속
2. **Sign up** 클릭
3. 이메일, 비밀번호, 사용자이름 입력 → 계정 생성
4. 이메일 인증 완료

---

## Step 4: 워크스페이스 다운로드

### 방법 A: Git으로 클론 (추천)

Git Bash 또는 터미널에서:

```bash
cd ~/Desktop
git clone https://github.com/kyjksw-cyber/iloom-workspace-codex.git
cd iloom-workspace-codex
```

바탕화면에 `iloom-workspace-codex` 폴더가 생깁니다.

### 방법 B: ZIP으로 다운로드 (Git이 안 될 때)

1. https://github.com/kyjksw-cyber/iloom-workspace-codex 접속
2. 초록색 **"<> Code"** 버튼 클릭
3. **"Download ZIP"** 클릭
4. 다운받은 ZIP을 바탕화면에 압축 해제
5. 폴더 이름을 `iloom-workspace-codex`로 변경

⚠️ ZIP으로 받으면 숙제 제출(push)이 안 됩니다. 가능하면 방법 A를 쓰세요.

---

## Step 5: AGENTS.md 본인 정보로 수정

`iloom-workspace-codex` 폴더 안의 `AGENTS.md` 파일을 메모장이나 VS Code로 열어서 상단을 수정:

```markdown
## 사용자 프로필
- **이름**: 홍길동
- **소속**: 고문정팀 리서치
- **주요 업무**: 소비자 조사, 리뷰 분석
```

저장하면 끝. 이제 Codex가 본인 정보를 알고 일합니다.

---

## Step 6: Codex CLI 실행

워크스페이스 폴더에서:

```bash
cd ~/Desktop/iloom-workspace-codex
codex
```

프롬프트가 뜨면 성공! 이제 자연어로 요청하면 됩니다.

# Part 3. 실습 — Codex로 데일리 노트 만들기

---

## 데일리 노트 만들기

```
> 데일리 노트 만들어줘
```

Codex가 AGENTS.md의 워크플로우대로:
1. 오늘 날짜 확인
2. 템플릿 읽기
3. `40-personal/41-daily/2026-03-20.md` 자동 생성

### 내용 추가하기

```
> 오늘 한 일에 "AX 1.5차 강의 수강, Codex CLI 실습" 추가해줘
```

### 작업 끝나고

```
> 오늘 한 일 정리해서 데일리 노트에 추가해줘
```

---

## 프로젝트 맥락 파일 만들기

### 새 프로젝트 시작

```
> 새 프로젝트 만들어줘. 이름: 봄 SNS 캠페인
```

자동 생성:
```
10-projects/15-봄-sns-캠페인/
├── AGENTS.md        ← 프로젝트 맥락
├── progress.md      ← 진행 현황
└── memo.md          ← 작업 메모
```

### 작업 후 기록

```
> progress.md 업데이트해줘
```

### 다음 세션에서 이어하기

```
> 봄 캠페인 이어서 할게. progress.md 먼저 읽어줘.
```

# Part 4. 숙제 — 데일리 노트 5일 챌린지

## 해야 할 것

**다음 강의 전까지, 매일 데일리 노트를 만드세요.**

### 규칙

1. 매일 Codex를 열고 `"데일리 노트 만들어줘"` 요청
2. 최소 **"오늘 한 일"** 과 **"내일 할 일"** 두 섹션만 채우면 OK
3. 업무 관련이든, 개인 일정이든 뭐든 상관없음
4. **5일치 완성**이 목표

### 완성하면 이렇게 됩니다

```
40-personal/41-daily/
├── 2026-03-24.md
├── 2026-03-25.md
├── 2026-03-26.md
├── 2026-03-27.md
└── 2026-03-28.md
```

## 제출 방법

### Git Bash 또는 터미널에서:

```bash
cd ~/Desktop/iloom-workspace-codex
git add -A
git commit -m "데일리노트 5일 챌린지"
git push
```

### push가 처음이라 에러가 나면:

GitHub 로그인 팝업이 뜰 수 있습니다 → 본인 GitHub 계정으로 로그인하면 됩니다.

만약 push 권한 에러가 나면:
1. GitHub에서 본인 계정으로 로그인
2. https://github.com/kyjksw-cyber/iloom-workspace-codex 에서 **Fork** 버튼 클릭
3. 본인 레포로 fork된 주소로 다시 clone:
```bash
git clone https://github.com/[본인아이디]/iloom-workspace-codex.git
```

### push가 정말 안 되면:

`40-personal/41-daily/` 폴더의 5개 파일을 **슬랙으로 보내주세요.** 괜찮습니다!
