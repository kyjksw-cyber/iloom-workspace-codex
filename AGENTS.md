## 작업 규칙

### Codex CLI 사용 시
1. **Ctrl+C**: 작업 중단
2. **새 세션**: 이전 맥락 초기화됨
3. **파일 참조**: 경로를 직접 언급 (예: "40-personal/41-daily/ 폴더의 파일 읽어줘")

### 파일 작업
- 기존 파일 편집 우선 (새 파일 생성 최소화)
- 날짜 형식: `YYYY-MM-DD_제목.md`
- 폴더 이동 시 경로 확인

### Git 규칙
- 작은 단위로 자주 커밋
- 커밋 메시지는 한글로 간결하게
- push 전 확인

## 활성 프로젝트

> **중요**: 프로젝트 작업 전 반드시 해당 프로젝트의 `progress.md`를 먼저 읽을 것.
> 각 프로젝트 폴더에도 `AGENTS.md`가 있으면 Codex가 자동으로 읽습니다.

| 프로젝트 | 맥락 파일 | 설명 |
|---------|----------|------|
| (예시) SNS 캠페인 | `10-projects/11-sns-campaign/progress.md` | 봄 캠페인 콘텐츠 기획 |

### 프로젝트 시작하기

새 프로젝트를 만들 때 Codex에게 이렇게 요청하세요:

```
"새 프로젝트 만들어줘. 이름: [프로젝트명]"
```

아래 구조로 자동 생성할 것:

```
10-projects/[번호]-[프로젝트명]/
├── AGENTS.md        ← 이 프로젝트 전용 맥락 (Codex 자동 인식)
├── progress.md      ← 진행 현황 (매 작업 후 업데이트)
└── memo.md          ← 작업 메모, 회의록 등
```

### 프로젝트 AGENTS.md 템플릿

```markdown
# [프로젝트명]

## 현재 상태
- (진행 중인 것)
- (완료된 것)
- (다음 할 것)

## 핵심 맥락
- 목표: 
- 타겟: 
- 기한: 

## 작업할 때 읽어야 할 파일
- progress.md — 진행 현황
- (기타 참고 파일)
```

### progress.md 작성 규칙

매 작업 후 progress.md를 업데이트할 것. 형식:

```markdown
# [프로젝트명] Progress

## 최근 업데이트
- YYYY-MM-DD: (한 일 요약)

## 완료
- [x] (완료된 작업)

## 진행 중
- [ ] (현재 작업)

## 다음 할 일
- [ ] (예정된 작업)
```

## 워크플로우

> 아래 워크플로우는 자연어로 요청하면 실행됩니다.

### 1. 데일리 노트

**요청**: "데일리 노트 만들어줘"

1. 오늘 날짜 확인 (YYYY-MM-DD)
2. 템플릿 읽기: `00-system/01-templates/daily-note-template.md`
3. 변수 치환 (date, weekday, yesterday, tomorrow, week)
4. 저장: `40-personal/41-daily/YYYY-MM-DD.md`
5. 이미 있으면 내용 표시 후 추가 작성 여부 확인

### 2. SNS 콘텐츠 초안

**요청**: "인스타 포스트 초안 만들어줘", "유튜브 쇼츠 스크립트 써줘", "블로그 글 초안 만들어줘"

**채널별 형식:**
- **Instagram**: 캡션(200자) + 해시태그(5-10개) + 이미지 가이드 + 캐러셀 구성
- **YouTube**: 썸네일 제목(3옵션) + Shorts 스크립트(60초) + 태그
- **Blog**: 제목(3옵션) + 서론/본론/결론 구조(1,500-2,000자) + SEO 키워드

**톤앤매너**: 일룸 브랜드 가이드 기본 적용
- 기본 해시태그: #일룸 #iloom
- 금지 표현: "최저가", "땡처리"
- 톤 키워드: 따뜻한, 편안한, 모던한

### 3. 브랜드 모니터링

**요청**: "일룸 모니터링 해줘", "경쟁사 동향 체크해줘"

1. 키워드 검색: 일룸, iloom, 한샘, 리바트, 에몬스
2. 소스: 네이버 블로그/카페, 커뮤니티, 뉴스
3. 감정 분석: 긍정/부정/중립 분류
4. 주제 분류: 제품 품질, 서비스, 가격, 경쟁사 비교
5. 리포트 생성: Executive Summary + 상세 분석 + Action Items

### 4. 씽킹 파트너

**요청**: "생각 정리 좀 도와줘", "브레인스토밍 해줘"

**행동 규칙:**
- 답하기 전에 질문으로 시작 — 3~5개 명확화 질문
- 인사이트 추적 — 핵심 발견과 연결점 기록
- 성급한 해결 금지 — 명시적 요청 전까지 탐색 모드 유지
- 아이디어 연결 — 패턴과 관계 식별
- 가정 표면화 — 암묵적 믿음 부드럽게 도전

### 5. 아이디어 캡처

**요청**: "아이디어 저장해줘", "이거 인사이트로 기록해줘"

1. 대화 내용에서 핵심 아이디어 추출
2. 카테고리 자동 분류 (business/branding/ai/marketing/product/daily/random)
3. 구조화된 문서 생성 (맥락 + 실무 적용 가능성 + 후속 행동)
4. 저장: `30-knowledge/YYYY-MM-DD_제목.md`

### 6. 할 일 관리

**요청**: "할 일 추가해줘", "투두 보여줘", "오늘 할 일 뭐야"

**추가** — `40-personal/46-todos/active-todos.md`에 추가:
```markdown
- [ ] [내용]
  - added: YYYY-MM-DD HH:mm
  - priority: [high/normal/low]
  - project: [프로젝트명]
```

**조회 옵션:** 전체 보기, 오늘 할 일, 프로젝트별, 오래된 것, 통계

### 7. 리프레임

**요청**: "내 요청 다시 정리해줘", "내가 뭘 원하는 건지 정리해줘"

대화 내용을 분석해서 구조화:
- **상황** → **핵심 목표** → **요구사항** → **제약사항** → **성공 기준** → **가정**

확인 후 다음 행동 제안.
