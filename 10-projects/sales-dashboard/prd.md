# 일룸 매출 통합 대시보드 PRD

## 1. 프로젝트 개요

### 배경

일룸은 오프라인 매장 15곳과 온라인 3개 채널(네이버 스마트스토어, 오늘의집, 공식몰)에서 가구를 판매한다. 현재 매출 데이터는 채널마다 포맷이 다르고, 상품코드 체계가 분리되어 있어 통합 분석에 매번 수작업이 필요하다.

**현재 Pain Points:**
- ERP에서 엑셀을 내려받아 직접 정리하고, 구글시트 연결에서 막힘
- 태블로로 1차 가공 후 분석 도구에 넣는 이중 작업
- 네이버/오늘의집/공식몰 데이터 포맷이 전부 달라 통합이 복잡
- 같은 상품인데 채널별 코드가 3종류 (NV-/OH-/내부코드)
- 매출만 보고 수익성(채산)은 별도 분석 필요
- 매번 자료 가공에 2~4시간 소요

### 목적

"데이터 넣으면 바로 보인다" - 원본 데이터 가공부터 대시보드 표시까지 전 과정을 자동화한다.

### 핵심 가치

1. **통합**: 4개 채널 매출을 하나의 화면에서 비교
2. **자동화**: 원본 파일 → 정제 → 시트 → 대시보드 파이프라인을 스킬 한 줄로 실행
3. **수익성**: 단순 매출이 아닌 채널별 마진/수수료를 반영한 채산 분석
4. **반복성**: 매월 새 데이터가 들어와도 같은 프로세스로 즉시 반영

## 2. 데이터 소스

모든 원본 데이터는 `50-resources/raw-data/`에 위치한다.

### 2.1 오프라인: ERP 수주 데이터

| 항목 | 값 |
|------|-----|
| 파일 | `erp-매출-YYYYMM.xlsx` (월별 분리) |
| 시트명 | 수주현황 |
| 행 수 | 50~60건/월 |
| 컬럼 수 | 20개 |
| 특이사항 | Row 1: 제목, Row 2: 출력일시, Row 3: 빈 행, Row 4-5: 병합 헤더 2줄, 마지막 행: 합계 |

**컬럼 (Row 5 기준):**
순번, 수주번호, 수주일자, 대리점코드, 품목코드, 품목명, 시리즈, 카테고리, 수량, 단가, 공급가, 소비자가, 할인율, 수주금액, 배송상태, 납기요청일, 배송예정일, 설치완료일, 결제방법, 비고

**대시보드에 필요한 컬럼:** 수주일자, 대리점코드, 품목코드, 품목명, 시리즈, 카테고리, 수량, 단가(공급가), 소비자가, 할인율, 수주금액, 배송상태

**정제 규칙:**
- Row 1~3 제거 (제목/출력일/빈행)
- Row 4~5 병합 헤더 → Row 5를 컬럼명으로 사용
- 마지막 합계 행 제거 (순번이 숫자가 아닌 행)
- 배송상태가 "취소"인 행 제외

### 2.2 온라인: 네이버 스마트스토어

| 항목 | 값 |
|------|-----|
| 파일 | `네이버-주문내역-YYYYMM.xlsx` |
| 시트명 | 주문조회 |
| 행 수 | ~80건/월 |
| 컬럼 수 | 30개 |
| 날짜 포맷 | `2026-01-24 21:10:00` (datetime) |
| 상품코드 | NV- prefix (예: NV-LP1200) |
| 금액 | 숫자 |
| 수수료 | ~5.5% (수수료 컬럼 존재) |

**대시보드에 필요한 컬럼:** 주문일시, 상품번호(NV코드), 상품명, 수량, 상품별 합계금액, 정산예정금액, 수수료, 주문상태

**정제 규칙:**
- 30개 컬럼 중 필요한 8개만 추출
- 주문일시에서 날짜만 추출 (시간 제거)
- 주문상태가 "취소" 관련인 행 제외
- 상품번호(NV코드) → 상품기준시트로 일룸상품코드 매핑

### 2.3 온라인: 오늘의집

| 항목 | 값 |
|------|-----|
| 파일 | `오늘의집-주문현황-YYYYMM.xlsx` |
| 시트명 | 주문현황 |
| 행 수 | ~45건/월 |
| 컬럼 수 | 20개 |
| 날짜 포맷 | `2026.01.20` (점 구분) |
| 상품코드 | OH- prefix (예: OH-LP1200) |
| 금액 | **텍스트** ("1,890,000" - 쉼표 포함 문자열) |
| 수수료 | 10~15% (카테고리별, 수수료율 컬럼 존재) |

**대시보드에 필요한 컬럼:** 주문일, 셀러상품코드(OH코드), 노출상품명, 주문수량, 결제금액, 정산금액, 수수료율, 배송상태

**정제 규칙:**
- 금액 컬럼(판매가, 할인금액, 결제금액, 정산금액) 텍스트 → 숫자 변환 (쉼표 제거)
- 수수료율 텍스트("15%") → 숫자(0.15) 변환
- 날짜 점(.) → 하이픈(-) 통일
- 셀러상품코드(OH코드) → 상품기준시트로 일룸상품코드 매핑

### 2.4 온라인: 공식몰

| 항목 | 값 |
|------|-----|
| 파일 | `공식몰-주문-YYYYMM.csv` |
| 행 수 | ~35건/월 |
| 컬럼 수 | 15개 |
| 컬럼명 | **영문** (order_id, order_date, product_code 등) |
| 날짜 포맷 | `MM/DD/YYYY` (미국식) |
| 상품코드 | 일룸 내부코드 그대로 (LP-1200) |
| 금액 | 숫자 |
| 수수료 | 없음 (자사몰) |

**컬럼:** order_id, order_date, product_code, product_name, option, qty, unit_price, total_price, discount, payment, shipping_fee, customer_name, address, status, payment_method

**대시보드에 필요한 컬럼:** order_date, product_code, product_name, qty, unit_price, total_price, discount, status

**정제 규칙:**
- 날짜 MM/DD/YYYY → YYYY-MM-DD 변환
- 영문 컬럼명 → 한글 통일 (또는 통합 테이블의 표준 컬럼명으로)
- status가 "cancelled"인 행 제외
- 주소에 쉼표 포함 가능 (CSV 파싱 주의)

### 2.5 기준 데이터: 상품기준시트

| 항목 | 값 |
|------|-----|
| 파일 | `상품기준시트.xlsx` |
| 시트명 | 상품마스터 |
| 행 수 | 13개 SKU (공통 10 + 온라인전용 3) |
| 컬럼 | 일룸상품코드, 상품명, 시리즈, 카테고리, 소비자가, 네이버상품번호, 오늘의집셀러코드, 공식몰코드, 등록상태 |

**핵심 역할:** 채널별 상품코드를 일룸상품코드로 통합 매핑
- NV-LP1200 → LP-1200
- OH-LP1200 → LP-1200
- 공식몰은 이미 일룸코드 사용

**온라인 전용 상품 3개 (ERP에 없음):**

| 일룸코드 | 상품명 | 카테고리 | 소비자가 |
|----------|--------|----------|----------|
| MN-QM | 모나코 퀸 매트리스 | 침실 | 690,000 |
| PT-D120 | 포테 데스크 1200 | 서재 | 359,000 |
| SN-SC | 시에나 사이드체어 | 거실 | 189,000 |

### 2.6 기준 데이터: 매장마스터

| 항목 | 값 |
|------|-----|
| 파일 | `매장마스터.xlsx` |
| 시트명 | 매장목록 |
| 행 수 | 15개 매장 |
| 컬럼 | 대리점코드, 매장명, 지역, 매장유형, 개설일, 면적(평), 담당자 |

**지역 분포:** 서울 3, 경기 4, 부산 2, 대구 1, 인천 1, 광주 1, 대전 1, 제주 1, 강원 1

### 2.7 목표실적

| 항목 | 값 |
|------|-----|
| 파일 | `목표실적.csv` |
| 형태 | **피벗** (월이 가로 배치) |
| 컬럼 | 매장코드, 2026-01, 2026-02, 2026-03 |
| 주의 | "매장코드" (ERP는 "대리점코드" - 동일한 값, 컬럼명만 다름) |

**정제 규칙:**
- 피벗 → unpivot 변환 (매장코드, 월, 목표금액)
- "매장코드" → "대리점코드"로 컬럼명 통일

## 3. 데이터 파이프라인

### 3.1 전체 흐름

```
원본 파일 (raw-data/)
    ↓
[Step 1] 구조 파악 & 정제
    - ERP: 병합헤더/합계행 제거, 취소 제외
    - 네이버: 30→8컬럼 추출, 날짜 정리
    - 오늘의집: 텍스트→숫자, 날짜 통일
    - 공식몰: 날짜 변환, 취소 제외
    ↓
[Step 2] 코드 매핑
    - 상품기준시트 기준으로 채널별 코드 → 일룸상품코드
    - NV-LP1200 / OH-LP1200 / LP-1200 → LP-1200
    ↓
[Step 3] 채널 통합
    - 4개 소스를 단일 테이블로 병합
    - 통합 컬럼: 날짜, 채널, 일룸상품코드, 상품명, 시리즈,
      카테고리, 수량, 매출금액, 공급가, 수수료, 정산금액, 지역, 매장명
    - 채널 구분: ERP/네이버/오늘의집/공식몰
    ↓
[Step 4] 매장/목표 조인
    - 대리점코드 → 매장마스터 조인 (매장명, 지역)
    - 목표실적 unpivot 후 조인 (오프라인만)
    ↓
[Step 5] 구글시트 입력
    - 가공 완료 데이터를 구글시트에 기록
    - 시트 구조: 통합매출, 매장별요약, 채널별요약, 목표달성
    ↓
[Step 6] 대시보드 반영
    - 시트 데이터를 대시보드 API로 fetch
    - 브라우저 새로고침으로 반영
```

### 3.2 통합 테이블 스키마

가공 완료 후 최종 통합 테이블의 컬럼:

| 컬럼명 | 타입 | 출처 | 설명 |
|--------|------|------|------|
| date | date | 각 채널 날짜 통일 | YYYY-MM-DD |
| month | string | date에서 파생 | YYYY-MM |
| channel | string | 소스 구분 | ERP/네이버/오늘의집/공식몰 |
| product_code | string | 상품기준시트 매핑 | 일룸상품코드 (LP-1200) |
| product_name | string | 상품기준시트 | 상품명 |
| series | string | 상품기준시트 | 시리즈 |
| category | string | 상품기준시트 | 카테고리 |
| qty | number | 각 채널 | 수량 |
| sale_amount | number | 매출금액 | ERP: 수주금액, 네이버: 합계금액, 오늘의집: 결제금액, 공식몰: total_price |
| supply_price | number | 공급가 | ERP: 공급가, 온라인: 소비자가 기준 |
| commission | number | 수수료 | ERP: 0, 네이버: ~5.5%, 오늘의집: 10-15%, 공식몰: 0 |
| settled_amount | number | 정산금액 | 매출 - 수수료 (실제 입금액) |
| margin | number | 계산 | settled_amount - supply_price (채산) |
| margin_rate | number | 계산 | margin / sale_amount * 100 |
| store_code | string | ERP만 | 대리점코드 (온라인은 null) |
| store_name | string | 매장마스터 조인 | 매장명 (온라인은 채널명) |
| region | string | 매장마스터 조인 | 지역 (온라인은 "온라인") |

### 3.3 채산(수익성) 계산 근거

| 채널 | 매출금액 | 수수료 | 정산금액 | 공급가 출처 | 마진 |
|------|----------|--------|----------|-------------|------|
| ERP(오프라인) | 수주금액 | 0 | = 수주금액 | 단가(공급가) 컬럼 | 수주금액 - 공급가 |
| 네이버 | 상품별 합계금액 | 수수료 컬럼 (~5.5%) | 정산예정금액 | 상품기준시트 소비자가 x 원가율 | 정산예정금액 - 추정원가 |
| 오늘의집 | 결제금액 | 수수료율 x 결제금액 (10-15%) | 정산금액 | 상품기준시트 소비자가 x 원가율 | 정산금액 - 추정원가 |
| 공식몰 | total_price | 0 (자사몰) | = total_price | unit_price | total_price - unit_price |

**핵심 인사이트**: "같은 제품을 어디서 팔면 마진이 가장 좋은가?" 채널별 수익률 비교가 가능하다.

### 3.4 스킬 저장

전체 파이프라인을 Claude Code 스킬(`.claude/skills/`)로 저장하여 반복 실행 가능하게 한다.

실행 예시:
```
새 월별 데이터가 raw-data/에 추가되었어.
데이터 가공 스킬 실행해서 시트 업데이트하고 대시보드에 반영해줘.
```

## 4. 대시보드 화면

### 4.1 View 1: 채널별 매출 요약 (메인)

**목적:** 오프라인/온라인 전체 매출을 한눈에 파악

**구성:**
- 상단 KPI 카드 4개: 총 매출, 오프라인 매출, 온라인 매출, 온라인 비중(%)
- 채널별 매출 막대 차트 (ERP / 네이버 / 오늘의집 / 공식몰)
- 월별 추이 꺾은선 차트 (채널별 색상 구분)
- 기간 필터 (월 선택)

### 4.2 View 2: 매장별 실적 + 목표 달성률

**목적:** 오프라인 매장별 실적을 목표 대비로 평가

**구성:**
- 매장별 실적 vs 목표 수평 바 차트 (달성률 % 표시)
- 달성률 기준 색상: 100%+ 초록, 80-100% 노랑, 80% 미만 빨강
- 매장 유형별 필터 (프리미엄샵 / 브랜드샵)
- 상위 3 / 하위 3 매장 하이라이트
- 매장별 상세 테이블 (매출, 목표, 달성률, 전월 대비)

### 4.3 View 3: 지역별 비교

**목적:** 권역별 매출 분포와 트렌드 파악

**구성:**
- 지역별 매출 도넛 차트 (서울/경기/부산/기타)
- 지역별 월간 추이 차트
- 지역-매장 드릴다운 테이블

### 4.4 View 4: 상품/카테고리 분석

**목적:** 어떤 상품이 잘 팔리는지, 카테고리별 비중 파악

**구성:**
- 카테고리별 매출 비중 (학생방/침실/거실/서재)
- 상품별 판매 순위 Top 10
- 시리즈별 채널 분포 (어떤 시리즈가 온라인에서 더 잘 팔리는가)
- 온라인 전용 상품 3개의 실적 별도 표시

### 4.5 View 5: 채산 분석 (수익성)

**목적:** 매출뿐 아니라 실제 수익성을 채널별로 비교

**구성:**
- 채널별 마진율 비교 차트
- 상품별 채널 간 수익성 비교 (같은 제품의 채널별 마진)
- 수수료 영향 분석 (네이버 5.5% vs 오늘의집 10-15% vs 공식몰 0%)
- 채널별 정산금액 vs 매출금액 갭 시각화
- 수익성 기준 상품 순위 (마진 기준 Top/Bottom)

## 6. 비기능 요구사항

### 반응형

- 데스크탑 (1920px): 차트 2열 배치, 사이드 네비게이션
- 태블릿 (768px): 차트 1열, 상단 네비게이션
- 모바일 (390px): KPI 카드 스크롤, 차트 풀폭

### 데이터 갱신

- 구글시트 기반: 시트 데이터 변경 → API fetch → 새로고침 반영
- JSON 폴백: 시트 연결 불가 시 `public/data/` JSON 파일 사용
- 갱신 주기: 수동 (스킬 실행 + 새로고침)

### 성능

- 초기 로딩: 3초 이내
- 차트 렌더링: 1초 이내
- 데이터 규모: 월 200~250건 x 12개월 = 최대 3,000건 (성능 이슈 없음)
